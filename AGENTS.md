# AGENTS.md — Arize Python SDK v8

Modern Python SDK (`arize` v8.x) for the Arize AI platform. Uses `uv` as the package manager, `ruff` for linting/formatting, `mypy` for type checking, and `pytest` for tests. All tooling is configured in `pyproject.toml` and invoked via `taskipy`.

## Key Structure

```
sdk/python/arize/v8/
├── pyproject.toml                # All tool config (ruff, mypy, pytest, taskipy, coverage)
├── uv.lock                       # Dependency lock file — commit changes to this
├── src/arize/
│   ├── client.py                 # ArizeClient — main user-facing entry point
│   ├── config.py                 # SDKConfiguration
│   ├── _lazy.py                  # LazySubclientsMixin — register new sub-clients here
│   ├── _client_factory.py        # Client construction helper
│   ├── regions.py                # Region enum
│   ├── prompts/client.py         # Sub-client pattern — each domain has client.py
│   ├── spans/client.py
│   ├── datasets/client.py
│   ├── experiments/client.py
│   ├── projects/client.py
│   ├── spaces/client.py
│   ├── api_keys/client.py
│   ├── ai_integrations/client.py
│   ├── annotation_configs/client.py
│   ├── embeddings/               # Optional NLP/CV/tabular embedding generators
│   ├── _generated/               # ⚠ AUTO-GENERATED — never edit by hand
│   │   ├── api_client/           # OpenAPI-generated REST client (models, stubs, tests)
│   │   └── protocol/             # Protobuf-generated code
│   └── _flight/client.py         # Arrow Flight high-performance data transfer
├── tests/
│   └── unit/                     # Unit tests — mirrors src/arize/ structure
│       ├── conftest.py           # Shared fixtures (mock_sdk_config, mock_arize_client, etc.)
│       └── <module>/test_client.py
└── docs/                         # Sphinx documentation (auto-built on ReadTheDocs)
```

## Development Commands

Run from `sdk/python/arize/v8/`:

```bash
task lint         # ruff format + ruff check --fix (auto-corrects)
task type-check   # mypy static type analysis
task test         # pytest with branch coverage (90% minimum enforced)
```

Integration tests, environment variables, and CI task variants are documented in [DEVELOPMENT.md](DEVELOPMENT.md).

**Always run `task lint`, `task type-check`, and `task test` after completing a significant feature or refactor.**

After making significant changes, run the `arize-code-review` subagent (`.agents/agents/arize-code-review.md`) for a staff-level review before presenting results.

CI-only variants (no auto-fix): `task ci-format`, `task ci-lint`, `task ci-type-check`, `task ci-test`.

## Generated Code — Do Not Edit

| Path | Regenerated by |
|------|---------------|
| `src/arize/_generated/api_client/` | `./scripts/recompile_openapi.sh` (repo root) |
| `src/arize/_generated/protocol/` | `./scripts/recompile_protos.sh` (repo root) |

Ruff and mypy automatically exclude `_generated/` and `*_pb2.py` files.

## Conventions

- **Line length:** 80 chars (code), 110 chars (docstrings/pycodestyle)
- **Docstrings:** Google-style required for all public APIs
- **Imports:** Managed by ruff/isort — do not sort manually
- **Sub-clients:** Each domain (`prompts/`, `spans/`, etc.) has a `client.py` exposing a `*Client` class. Register new sub-clients in `_lazy.py` using the `LazySubclientsMixin` pattern.
- **Exceptions in docstrings:** Refer to `ApiException` (not the fully qualified `arize._generated.api_client.exceptions.ApiException`)
- **Test fixtures:** Add shared fixtures to `tests/unit/conftest.py`. Per-module fixtures go in `tests/unit/<module>/conftest.py`.
- **Coverage exclusions:** Legacy modules (`datasets`, `experiments`, `ml`, `projects`, `spans`) and generated code are excluded from the 90% coverage threshold. New modules must meet the threshold.
- **Python version:** Targets >=3.10; avoid syntax not available in 3.10.
